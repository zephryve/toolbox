Claude Code Session 标题中文化

方案 · 2026-06-14 · 跃西风 Zephryve · 机制全部实测验证

一句话：往 session jsonl 追加一条中文 custom-title 记录，picker 即显示中文，且设一次就锁死、永不随对话改动而变——因为 custom-title 优先级压过 AI 自动生成的 ai-title。

壹现象与目标

中文用户（首条消息、全程对话都中文）的 session 列表里，标题大多是英文，且改了名还担心"会不会又变回去"：

picker 现在显示想要
Plan Zephryve Engine architecture and workflow规划 Zephryve 飞轮架构
Scrape Alibaba job listings from career site抓取阿里招聘岗位
Design scoring system for job evaluation results岗位评估打分系统设计

目标两条：① 标题中文 ② 不老变（不能因为对话又更新了、标题又跳）。

贰机制原理（实测，非推断）

session jsonl 里有两种标题记录，谁压谁是关键：

ai-title（aiTitle）
Claude Code 自动生成 · 默认英文 · 随对话反复重写（实测一个 session 写了 715 次）
被压 ↓
custom-title（customTitle）
用户设定（GUI 改名 / 本工具）· 优先级更高 · 写入即锁定

三个实测铁证

① ai-title 默认英文，跟对话语言无关

抽样 8 个 session：首条消息 100% 中文，aiTitle 100% 英文；全库搜"中文 aiTitle"零命中。settings.json 的 language 配置只管 Claude 回复语言/语音听写，不管标题（官方文档确认无标题语言配置项）。

② custom-title 压 ai-title，写一次锁死

session c11c6a52：ai-title × 715（英文 Web scraping CRS） + custom-title × 715（中文 抓取CRS中外合作办学网页） → picker 显示中文。ai-title 重写 715 次都盖不过。

③ 外部写入 picker 认

往已关闭 session 0fc67be7（原 "Update Claude CLI"）jsonl 追加 custom-title "升级 Claude CLI" → 刷新 picker → 标题变中文 ✅（本方案前已实验通过）。

叁方案

工具只做一件事：给「有英文 ai-title 但无 custom-title」的 session 追加一条中文 custom-title。

  • 幂等——已有 custom-title 的跳过，每个标题只中文化一次，写入即锁定，重复跑无副作用。这就是"不老变"的保证。
  • 只追加不改原内容——往 jsonl 末尾加一行，绝不动原有任何行。
  • 写前整目录备份——备份到 ~/.claude/.session-title-backups/{时间戳}/，picker 异常可整目录还原。

两步走

步骤做什么状态
存量批量--apply 扫所有 session，把英文 ai-title 翻成中文、追加 custom-title（claude -p 批量翻译）待你确认跑
增量自动挂 SessionEnd hook（不是 Stop）——新 session 结束时自动中文化（已关闭场景，实测安全）可选

为什么增量用 SessionEnd 不用 Stop

custom-title 是 Claude Code 内部维护的状态（每次保存同步重写）。对正活着的 session 从外部写，可能被下次保存覆盖。SessionEnd 时 session 已关闭、标题已定——跟实验的已关闭 session 同场景，写入安全。

肆用法

# 1. 只扫描统计（不写文件）
python3 session_title_zh.py --scan --project zephryve-engine

# 2. 翻译并写入（自动备份 + 幂等）
python3 session_title_zh.py --apply --project zephryve-engine

# 3. SessionEnd hook 模式（只处理当前 session）
python3 session_title_zh.py --hook
