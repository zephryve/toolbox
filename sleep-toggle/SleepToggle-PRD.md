# SleepToggle PRD

## 要解决什么问题

macOS 用户在工作中经常需要临时离开电脑——接水、上厕所、开会、换个房间——但同时后台有任务正在跑（文件下载、代码编译、远程同步、Claude Code 长任务等）。出于安全习惯或公司规范，离开时会合盖或按电源键锁屏。

问题在于：macOS 合盖/按电源键后系统进入睡眠，WiFi 断开，所有后台任务中断。回来一看，下载到一半断了，编译重头来，远程连接掉了。

底层有一条命令 `sudo pmset -a disablesleep 1` 可以禁止睡眠，但存在两个实际障碍：
1. 每次要开终端输命令，不方便
2. 用完容易忘记恢复（`disablesleep 0`），导致电脑长期不睡眠，持续耗电

## 帮用户完成什么动作

**一键切换"锁屏但不睡眠"的状态。**

用户在菜单栏点一下，电脑进入"强力模式"——之后无论合盖、按电源键还是系统自动休眠，都不会真正睡眠，WiFi 保持连接，后台任务继续跑。再点一下，恢复正常。

## 输入 / 输出 / 后续

| | 说明 |
|---|---|
| 输入 | 用户点击菜单栏图标，选择"切换" |
| 输出 | 系统睡眠策略立即切换，图标实时反馈当前状态 |
| 正常模式 | `bolt.circle`（空心圆闪电），系统默认色。合盖正常睡眠 |
| 强力模式 | `bolt.circle.fill`（实心圆闪电），亮黄色警示。合盖不睡眠，后台保持运行 |
| 退出 app | 自动恢复正常睡眠，防止遗忘 |
| 开机 | 自动启动，常驻菜单栏，无需手动打开 |

## 约束与失败处理

| 场景 | 处理方式 |
|---|---|
| sudo 免密未配置 | 切换时弹窗提示错误码，引导运行 install.sh |
| app 被杀或崩溃 | 启动时读取 `pmset -g` 获取系统实际状态，图标与系统同步 |
| 用户忘记关强力模式 | 退出 app 时自动恢复；系统重启后 disablesleep 自动失效 |
| 长期合盖插电运行 | macOS 硬件级热管理 + 优化电池充电，日常联网负载无风险 |
| 分享给朋友 | 发一个 .dmg 文件，拖入 Applications + 跑一次 install.sh 即可 |

## 技术方案

核心需求是"零依赖、拿到就能用"，所以选 Swift 原生开发，预编译为 arm64 二进制，打包成 .dmg 分发，对方不需要安装任何开发工具。

| 项目 | 选择 | 理由 |
|---|---|---|
| 语言 | Swift | 零依赖，编译成原生 .app，体积约 120KB |
| 架构 | arm64 only | Apple Silicon Mac 专用，2025 年 Intel Mac 占比极低 |
| 菜单栏 | NSStatusBar + NSMenu | macOS 原生 API，无第三方库 |
| 图标 | SF Symbols | 系统内置矢量图标，无需图片资源 |
| app 图标 | Swift 脚本生成 .icns | 深色背景 + 金黄闪电，编译时生成 |
| 睡眠控制 | `sudo pmset -a disablesleep 1/0` | macOS 标准电源管理命令 |
| 免密 | `/etc/sudoers.d/sleeptoggle` | 仅对 pmset 免密，最小权限原则 |
| 开机自启 | LaunchAgent plist | macOS 标准方式 |
| 分发 | .dmg（预编译 .app + install.sh） | 用户无需装 Xcode 或任何开发工具 |

## 安装流程

1. 下载 `SleepToggle.dmg`（123KB）
2. 打开 .dmg，将 `SleepToggle.app` 拖到 Applications
3. 终端执行 `bash /Volumes/SleepToggle/install.sh`
4. 输入一次管理员密码（配置 pmset 免密，一次性操作）
5. 自动配置开机自启并启动，菜单栏出现闪电图标，完成

## 验证方式

强力模式下运行 `ping` 带时间戳，合盖 30 秒再开盖，检查时间戳连续无跳跃、0% 丢包。已验证通过。
